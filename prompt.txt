🔄 F3X STATE RESILIENCE
lua
-- Before EVERY batch of operations:
if not F3X:Init() then
    -- Tool was unequipped or lost
    notify("F3X tool lost — re-equipping...")
    equipBuildingTool()
    task.wait(0.5)
    if not F3X:Init() then
        error("F3X tool unavailable — build paused")
    end
end
If tool is unequipped mid-build: pause, re-equip, re-init, resume from last successful part
If InvokeServer returns nil or errors: retry 3× with 0.2s backoff, then mark part as failed and continue
After every BatchSize parts: validate F3X tool still exists in Character

After build completes:
lua
-- Count parts that were successfully created via F3X
local builtCount = 0
for i, part in pairs(partMap) do
    if part and part:FindFirstAncestorOfClass("DataModel") then
        builtCount += 1
    end
end

local expected = #buildData.Parts
if builtCount < expected then
    notify(string.format("⚠️ Validation failed: %d/%d parts exist", builtCount, expected))
    -- Offer to retry failed parts
else
    notify(string.format("✅ Validation passed: all %d parts confirmed in workspace", builtCount))
end
